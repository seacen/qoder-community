# 产业链定位诊断分析

## 触发条件

当用户提交企业产业链定位文字描述，或要求诊断、审阅、优化专精特新"小巨人"申报书第二部分（所属产业链）时使用本技能。

## 输入

用户提供的企业产业链定位文字描述，通常为申报书"第二部分 所属产业链"的初稿或已有文本。

## 诊断维度

从以下12个维度对输入文字进行系统性诊断：

1. **产业链数量与覆盖**：是否在3-5条之间，是否属于重点产业链
2. **链条展开格式**：是否严格按"XX—>XX—>XX—>企业所在环节"逐级展开
3. **关键环节定位**：是否精确到细分领域的关键环节
4. **企业作用阐述**：是否清晰说明"做什么""怎么做"
5. **价值意义**：是否说明对整条产业链的实际价值
6. **补短板/锻长板/填空白**：是否体现强链补链稳链作用
7. **主导产品领域匹配**：是否符合制造业核心基础零部件、关键软件等重点领域
8. **数据真实性**：是否虚构产业链归属或夸大作用
9. **语言规范**：是否存在绝对化表述
10. **数据标注规范**：推算值是否标注来源
11. **结构完整性**：是否包含名称、链条、细分领域、关键环节、企业作用、价值意义
12. **技术关联性**：是否关联专利与I类知识产权

## 工作流程

按以下六个阶段执行：

1. **输入接收与预处理**：将用户文本保存至临时文件
2. **自动化分析**：调用诊断引擎执行规则化检查
   ```bash
   python scripts/diagnosis_engine.py --input input.txt --output report.md --format markdown
   ```
3. **结构合规审查**：核对产业链数量、链条展开格式与结构完整性
4. **内容深度审查**：检查关键环节精度、企业作用、价值意义与战略定位
5. **事实与语言审查**：核查主导产品领域匹配、数据真实性、语言规范
6. **问题汇总与输出**：基于引擎结果与专家视角生成最终诊断报告

## 输出格式

### 一、总体评价

用2-4句话概括整体质量，指出最突出的1-3个优点和最突出的1-3个不足。

### 二、问题诊断

分条列出发现的问题。每条包含：
- **问题类别**：对应上述12个诊断维度之一
- **严重程度**：【严重】/【一般】/【建议】
  - 【严重】：可能导致申报被质疑或扣分的关键缺陷
  - 【一般】：影响申报质量但非致命的问题
  - 【建议】：锦上添花类的优化空间
- **具体问题**：具体描述问题是什么
- **原文引用**：摘录原文中存在问题的具体句子或片段

### 三、修改建议

针对发现的问题，给出两到三条最具体、可操作的修改建议，说明"如何改""改成什么样"。

## 执行规则

- 当用户提交产业链定位文字时，先调用诊断引擎进行自动化分析，再结合专家视角输出诊断报告
- 若用户仅说"帮我诊断"但未提供文字，提示用户粘贴需要诊断的产业链定位文字
- 若用户提供的文字明显不是产业链定位描述，提示用户确认输入内容
- 诊断过程保持建设性、专业化语气
- 对基于合理推理得出的判断标注"【推算值，需补充权威来源】"
- 对信息缺失无法判断的情况标注"⚠️数据缺失，需补充[缺失项]"
